# Cobblestone Editor

Cobblestone is a project I created because Visual Studio Code wasn't quite to my liking. I wanted an editor with better features and a workflow that actually makes me happy to use it.

## Why Cobblestone?

- VS Code was missing basic usability. For example, the file explorer and search are separated, not integrated—why!?
- I wanted more built-in features and a streamlined experience out of the box.
- All telemetry and tracking have been completely stripped out for privacy.
- I will keep improving and adding things as I go. This is my vision for a better editor.

## Key Features & Improvements

- **Fully working build & debug control buttons** (run bar) — fast project cycles.
- **Modern code style management** — Profiles, save-participant, auto-fix on save, status bar indicator, and full UI.
- **Integrated NuGet package management** — Search, install, remove and manage .NET/NuGet packages right in the editor.
- **Unified search modal:** Fast keyboard shortcut, recent searches, all types (files, symbols, actions), and new UI.
- **Default icon and theme upgrades** — Brand new SVG logos, default icon sets (VS Minimal Icon Theme), and richer appearance.
- **Modals & dialogs upgraded** — True VS Code webview, accessibility (ARIA, Escape, tab/focus trap), and custom options.
- **Complete removal of telemetry** — No usage tracking, ever. All telemetry code is gone by default.
- **Refined file explorer and search experience** — Actually useful, more native feel.
- Continuous refactors for code style, clarity, and maintainability.

...and whatever else I dream up—there’s more every week.
